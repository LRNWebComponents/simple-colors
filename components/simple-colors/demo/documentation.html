simple-colors documentation

Getting started

To use simple-colors in a component:
  1. bower install --save LRNWebComponents/simple-colors
  2. Include <link rel="import" href="path/to/simple-colors/simple-colors.html">
  3. To allow an element to inherit the variables, add the behavior: simpleColorsBehaviors,
  4. Add custom styles to your element using the simple colors variables. For example:
    <style is="custom-style" import="simple-colors">
        :host {
            color: --simple-colors-accent-foreground1;
            --paper-listbox-background-color: --simple-colors-accent-background2;
            border-color: 1px solid var(--my-component-border-color, --simple-colors-red-background3);
        }
    </style>
  5. …Or use the simple-colors text, background and border color classes. For example:
    class="simple-colors-light-blue-foreground1-text simple-colors-light-blue-background5-border simple-colors-lime-background1"

simple-colors naming conventions

To see these naming conventions in action, see color reference.

CSS variable names

The naming convention for simple-colors CSS variables is: --simple-colors-(light-theme or dark-theme)-(color)-foreground or background contrast level. For example:

  • For the light theme's highest contrast indigo foreground color, use --simple-colors-light-theme-indigoforeground1
  • For the light theme's lowest contrast foreground color in the element's current accent color, use --simple-colors-light-theme-accent-foreground5
  • For the dark theme's lowest contrast default greyscale background color, use --simple-colors-dark-theme-background5
  • For the element's current theme's highest contrast lime background color, use --simple-colors-lime-background1

CSS class names

The naming convention for simple-colors CSS classes is: .simple-colors-(optional color)-foreground or background with contrast level-use case. Note that the classes do not allow you to specify a theme. For example:

  • For a default greyscale background-color with the highest possible contrast background, use --simple-colors-background5
  • For a text color with highest contrast indigo foreground, use --simple-colors-indigo-foreground1-text
  • For a border-color with the lowest contrast foreground in the element's current accent color, use --simple-colors-accent-foreground5-border

theme name

The theme name, is optional. It can be light-theme or dark-theme.

  • The --simple-colors-light-theme colors are based on a light background with a dark foreground.
  • The --simple-colors-dark-theme colors are based on a dark background with a light foreground.

If the theme name is omitted, the theme will be applied based on whether or not the element or its ancestor has a dark attribute. This allows you to specify only one set of CSS color variables and allow a single attribute to switch between light and dark. In other words, you can change --simple-colors-red-foreground1 from --simple-colors-light-theme-red-foreground1 to --simple-colors-dark-theme-red-foreground1simply by adding a dark attribute that uses simple colors behaviors.

color

The color, is optional. If omitted, the color will be in greyscale. Simple colors using the naming and similar shares of Materialize. However, colors have been adapted to support WCAG 2.0 AA contrast ratios. The colors availble are: red, pink, purple, deep-purple, indigo, blue, light-blue, cyan, teal, green, light-green, lime, yellow, amber, orange, deep-orange, brown, blue-grey.

There is also an accent color variable that changes based on an the accent-color attribute of an element using simple-colors behaviors. For example, if accent-color="indigo", then --simple-colors-accent-foreground2 will be set to --simple-colors-indigo-foreground2. If your accent color is indigo, but you want to use red for a delete button, you can either, set the button to accent-color="red" to override the parent's color, or you can use CSS to change its color to --simple-colors-red-background1.

foreground or background with level

For each theme there are 5 foreground and 5 background contrast values for each color in a theme. The lower the number, the higher the contrast. For example, foreground2 has a higher contrast than foreground4.

Most often you will want to use the highest-contrast foreground with the highest-contrast background to ensure readability. For short sections of text, such as callouts or UI buttons, you may reduce the contrast. Specific WCAG 2.0 Guidelines are below:

Small Text

Small text requires the highest contrast ratio for WCAG 2.0 AA compliance. Small text is any content that is NOT:

  • bold text that is 14 point or higher,
  • text that is 18 point or higher,
  • a decorative elements that has no semantic meaning, such as a border, or
  • a disabled UI element

Large Text

WCAG 2.0 AA requires higher contrast for small text than for large text and for UI elements that have no text. Large text is defined as:

  • bold text that is 14 point or higher, or
  • text that is 18 point or higher

Other UI Elements

WCAG 2.0 AA allows the following exceptions for contrast:

  • decorative elements that have no semantic meaning, such as borders
  • disabled UI elements
